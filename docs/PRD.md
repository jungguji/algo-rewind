# PRD: Algo-Rewind (Local File Web Edition)

---

### 1. 개요 (Overview)

-   **제품명**: Algo-Rewind
-   **제품 목표**: 알고리즘 문제 풀이 기록을 사용자의 로컬 컴퓨터에 **안전한 파일 형태**로 저장하고, 웹 브라우저를 통해 손쉽게 불러와 간격 반복(Spaced Repetition) 원리에 기반한 체계적인 복습을 돕는 개인용 웹 애플리케이션.
-   **핵심 가치**:
    -   **데이터 주권 및 안정성**: 모든 기록은 사용자의 컴퓨터에 `JSON` 파일로 직접 저장되므로, 브라우저 캐시 삭제나 서비스 중단으로부터 데이터가 100% 안전하다.
    -   **단순함과 접근성**: 별도의 설치나 회원가입 없이, 단일 HTML 파일을 브라우저에서 여는 것만으로 모든 기능을 즉시 사용할 수 있다.
    -   **능동적 학습 지원**: 문제별 '핵심 메모' 기능을 통해 단순 암기가 아닌, 문제 해결 과정의 사고를 복기하도록 유도하여 학습 효과를 극대화한다.

---

### 2. 사용자 (User Persona)

-   **대상**: 알고리즘 학습을 꾸준히 하며, 풀었던 문제들을 잊지 않고 온전히 자신의 것으로 만들고 싶어 하는 모든 학습자 (취준생, 개발자 등).
-   **요구사항(Needs)**:
    -   내가 기록한 소중한 데이터가 절대 사라지지 않았으면 좋겠다.
    -   복잡한 프로그램 설치 없이, 웹 브라우저만으로 간단히 사용하고 싶다.
    -   어떤 문제를 언제 복습해야 할지 자동으로 알려주었으면 좋겠다.
    -   문제별로 나의 풀이 아이디어나 실수했던 점을 메모하고 싶다.

---

### 3. 핵심 기능 (Core Features)

**화면 구성**: 직관적인 단일 페이지 애플리케이션(SPA)

**Feature 1: 데이터 관리 (Data Management - 가장 중요!)**
-   **📂 데이터 불러오기 (Import)**:
    -   페이지에 진입했을 때 가장 먼저 보이는 기능.
    -   사용자가 이 버튼을 클릭하여 로컬에 저장된 `algo-rewind.json` 파일을 선택하면, 파일의 내용을 읽어와 전체 데이터를 화면에 렌더링한다.
-   **💾 데이터 저장하기 (Export)**:
    -   페이지 상단에 항상 떠 있는 버튼.
    -   현재까지의 모든 변경사항(새 문제 등록, 복습 완료 처리 등)이 반영된 전체 데이터를 `algo-rewind.json` 파일로 사용자의 '다운로드' 폴더에 저장(덮어쓰기)한다.
    -   **사용자는 작업을 마칠 때 이 버튼을 눌러 데이터를 백업하는 습관을 가진다.**

**Feature 2: 문제 등록 (Add a Problem)**
-   페이지의 특정 영역(예: 상단)에 위치한 입력 폼.
-   **입력 필드**:
    -   `문제 이름/번호` (Text, 필수)
    -   `문제 링크` (Text)
    -   `태그` (Text, 쉼표로 구분)
    -   `핵심 메모` (Text Area, 문제 해결의 핵심 아이디어나 실수 기록)
    -   `나의 이해도` (Select/Radio: AGAIN, HARD, GOOD, EASY)
-   **동작**: '등록하기' 버튼 클릭 시, 입력된 정보와 계산된 `다음 복습일`이 화면의 목록에 즉시 추가된다. (실제 파일 저장은 '데이터 저장하기' 버튼을 눌렀을 때 일어남)

**Feature 3: 복습 대시보드 (Review Dashboard)**
-   **A. 🔔 오늘 복습할 문제 (Today's Review)**:
    -   데이터 로드 시, `다음 복습일`이 오늘이거나 과거인 문제들이 자동으로 필터링되어 최상단에 노출된다.
    -   각 문제 아이템은 `문제 이름`, `태그`, `핵심 메모 보기/숨기기 토글 버튼`으로 구성된다.
    -   **복습 완료 액션**:
        -   각 문제 아이템 옆의 '복습 완료' 버튼을 누른다.
        -   이해도를 다시 선택하는 간단한 팝업(Modal)이 뜬다.
        -   새로운 이해도를 선택하면, `다음 복습일`이 갱신되고 해당 문제는 '오늘 복습할 문제' 목록에서 사라진다.

-   **B. 📚 전체 문제 목록 (All Problems)**:
    -   '오늘 복습할 문제' 목록 아래에 위치하며, 기록된 모든 문제가 표시된다.
    -   **기능**: 이름/태그 기반의 간단한 `검색(필터)` 기능, `다음 복습일 순`, `최신 등록 순` 등 `정렬` 기능.

---

### 4. 복습 주기 로직 (SRS Logic)

-   사용자의 '이해도 평가'에 따라 다음 복습 날짜가 자동으로 계산된다.
-   **`AGAIN`**: + 1일
-   **`HARD`**: + 3일
-   **`GOOD`**: + 7일
-   **`EASY`**: + 30일

---

### 5. 기술 스택 및 데이터 구조

-   **언어**: Rust, Vanilla JavaScript, HTML, CSS (프레임워크 없음)
-   **실행 환경**: 모든 최신 웹 브라우저 (별도 서버 필요 없음)
-   **데이터 저장소**: 사용자의 로컬 파일 시스템 (**`JSON` 파일**)
-   **데이터 구조 (`algo-rewind.json` 파일의 내용)**:
    ```json
    [
      {
        "id": 1730510000000,
        "name": "BOJ 1000 A+B",
        "url": "https://www.acmicpc.net/problem/1000",
        "tags": ["#기초", "#구현"],
        "memo": "단순 입출력 문제. Scanner나 BufferedReader 사용법 익히기.",
        "level": "EASY",
        "createdAt": "2025-11-02",
        "nextReviewAt": "2025-12-02"
      }
    ]
    ```

### 6. 사용자 플로우 (User Flow)

1.  **시작**: 사용자가 로컬에 저장된 `index.html` 파일을 브라우저로 연다.
2.  **데이터 로드**: `📂 데이터 불러오기` 버튼을 눌러 `algo-rewind.json` 파일을 로드한다. (최초 사용 시에는 이 단계를 건너뛴다)
3.  **확인 및 복습**: '오늘 복습할 문제' 목록을 확인하고, 해당 문제들을 다시 풀어본다. 복습을 마친 문제는 '복습 완료' 처리를 한다.
4.  **기록**: 새로 푼 문제가 있다면 '문제 등록' 폼을 통해 추가한다.
5.  **저장**: **오늘의 모든 작업을 마친 후, `💾 데이터 저장하기` 버튼을 눌러 변경된 내용을 `algo-rewind.json` 파일에 덮어쓴다.** 이 파일은 사용자가 원하는 곳(예: Dropbox, Google Drive, Git Repository)에 백업할 수 있다.

---

### 7. MVP 범위에서 제외할 기능 (Out of Scope)

-   **자동 저장**: 사용자의 혼란을 막고 데이터 흐름을 명확히 하기 위해, 명시적인 저장/로드 기능에 집중한다.
-   **클라우드 동기화 및 로그인**: 모든 데이터는 로컬 파일 기반으로만 동작한다.
-   **복잡한 UI/UX**: 기능 구현에만 집중하며, 최소한의 CSS로 가독성만 확보한다.
